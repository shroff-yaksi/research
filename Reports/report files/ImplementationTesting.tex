\chapter{Implementation}

This chapter describes the RE-TabSyn pipeline: environment setup, data preprocessing, and the synthetic data generation procedure. Experimental design details --- datasets, model configurations, training procedures, and evaluation metrics --- are in Chapter~\ref{ch:experimental_design}.

All experiments were conducted using Python 3.13.5 with PyTorch 2.9.1, on a MacBook Air (Apple M3, 16 GB unified RAM, macOS 26.4). Supporting libraries: NumPy 2.1.3, Pandas 2.2.3, scikit-learn 1.6.1, XGBoost 3.1.2.

\section{Experimental Environment}

All experiments ran on a single machine with no distributed computing dependencies. The hardware and software stack are described below.

\subsection{Hardware}

All experiments ran on a MacBook Air (Apple M3, 16 GB unified RAM). The M3's unified memory architecture means the CPU and GPU share the same memory pool, removing the CPU-to-VRAM transfer overhead common in discrete GPU setups. The largest training run (RE-TabSyn on Polish Bankruptcy, 64 features) completes within a few hours.

\subsection{Software stack}

Python 3.13.5 is the primary language. PyTorch 2.9.1 was chosen for its eager execution model and consistency with the official TabSyn and TabDDPM implementations. NumPy 2.1.3 and Pandas 2.2.3 handle preprocessing and data loading. scikit-learn 1.6.1 provides the QuantileTransformer and LabelEncoder, fit on training data only to prevent leakage. XGBoost 3.1.2 \cite{chen2016xgboost} is the downstream classifier in all TSTR evaluations.

\section{End-to-End Pipeline Architecture}

Figure~\ref{fig:pipeline} shows the sequential flow from raw data ingestion to synthetic output. Each module has a well-defined input and output, so failures can be isolated to a specific stage.

\begin{figure}[H]
\centering
\includegraphics[width=0.28\textwidth]{img/pipeline.png}
\caption{RE-TabSyn end-to-end pipeline overview.}
\label{fig:pipeline}
\end{figure}

The pipeline has five modules:
\begin{enumerate}
    \item Data Ingestion: Unified access to all six benchmark datasets with a standardized interface that identifies numerical columns, categorical columns, and the target variable.

    \item Preprocessing: Imputation, encoding, and scaling applied to training data only, then identically to the test split.

    \item Model Training: Two-phase protocol --- VAE to convergence, then Latent Diffusion Model with CFG on frozen VAE latents.

    \item Generation: Latent sampling, CFG-guided denoising, VAE decoding, and reverse transformation. Output is a Pandas DataFrame matching the original data format.

    \item Evaluation: Fidelity, utility, and privacy metrics computed across three random seeds, reported as mean $\pm$ standard deviation.
\end{enumerate}

\section{Data Preprocessing}
\label{sec:preprocessing}

The preprocessing pipeline applies four steps in order: (1) missing value imputation, (2) feature encoding, (3) feature scaling, and (4) train-test splitting. All transformers are fit on training data only. Figure~\ref{fig:preprocessing_flow} shows the full flow.

\begin{figure}[H]
\centering
\includegraphics[width=0.55\textwidth]{img/preprocessing_flow.png}
\caption{Preprocessing: numerical and categorical branches.}
\label{fig:preprocessing_flow}
\end{figure}

\subsection{Missing value handling}

Numerical columns are imputed with the column median. The median is preferred over the mean for financial data because skewed distributions (a few very large transactions or incomes) pull the mean away from the typical value.

Categorical columns are imputed with the column mode. An alternative approach --- treating missingness as a distinct category --- is also supported when the pattern of missingness carries information.

\subsection{Feature encoding and scaling}

Numerical features are mapped to $\mathcal{N}(0, 1)$ using a Quantile Transformer. Financial distributions are almost never Gaussian: transaction amounts, for example, cluster around small values with a long right tail. The Quantile Transform is invertible and gives equal representational weight to every part of the distribution during training.

Categorical features are label-encoded to integers in $\{0, 1, \ldots, K-1\}$. One-hot encoding creates a high-dimensional, sparse binary vector per feature; the diffusion noise process produces dense vectors that are incompatible with this sparse structure. Label encoding avoids the dimensionality explosion, and the VAE encoder learns continuous embeddings from the integer-coded categories.

\section{Synthetic Data Generation Procedure}

Once trained, RE-TabSyn generates synthetic data in four steps without retraining.

\textbf{Step 1 --- Latent Initialization:} Sample $N$ initial noise vectors from $\mathcal{N}(0, I)$ in the 64-dimensional latent space. To generate balanced data, assign half the vectors class label $y = 1$ and half $y = 0$. Any target ratio can be set by adjusting this split.

\textbf{Step 2 --- CFG-Guided Denoising:} Run 1,000 reverse diffusion steps. At each step:
\begin{enumerate}
    \item Compute conditional noise: $\epsilon_{cond} = \epsilon_\theta(\mathbf{z}_t, t, y)$
    \item Compute unconditional noise: $\epsilon_{uncond} = \epsilon_\theta(\mathbf{z}_t, t, \emptyset)$
    \item Apply CFG: $\tilde{\epsilon} = \epsilon_{uncond} + w \cdot (\epsilon_{cond} - \epsilon_{uncond})$, with $w = 2.0$
    \item Update latent: $\mathbf{z}_{t-1} = f(\mathbf{z}_t, \tilde{\epsilon}, t)$
\end{enumerate}

\begin{figure}[H]
\centering
\includegraphics[width=0.5\textwidth]{img/cfg_denoising_step.png}
\caption{Single CFG denoising step, $w=2.0$.}
\label{fig:cfg_denoising_step}
\end{figure}

CFG requires two DiT forward passes per step, making generation roughly twice as slow as unconditional TabSyn. Fast sampling via DDIM \cite{ho2020ddpm} can reduce this to 50 steps with comparable quality.

\textbf{Step 3 --- VAE Decoding:} Pass denoised latent vectors through the frozen VAE decoder. Numerical features are produced directly from linear output heads. Categorical features are obtained via argmax over per-column softmax probabilities.

\textbf{Step 4 --- Reverse Transformation:} Apply the inverse preprocessing pipeline. Numerical features are mapped back via Inverse Quantile Transformer (an exact inverse preserving percentile rank). Categorical features are recovered by Inverse Label Encoding. The result is a Pandas DataFrame in the original feature space.
